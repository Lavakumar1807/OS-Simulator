Producer-Consumer Problem

The Producer-Consumer Problem is a classic synchronization problem in operating systems. It involves two types of processes:

  • Producers: Create data items and place them in a shared buffer.
  • Consumers: Take data items from the shared buffer for processing.

The challenge is to ensure proper synchronization between producers and consumers to avoid:

  • Buffer Overflow: Producers trying to add items to a full buffer.
  • Buffer Underflow: Consumers trying to remove items from an empty buffer.
  • Race Conditions: Simultaneous access to the buffer causing data corruption.

This simulation demonstrates how semaphores and mutual exclusion can solve these challenges.
